# 领域知识文档

> 本项目领域知识沉淀，用于记录约定、踩坑记录和代码模式。
> 由 /tech:init 初始化，/tech:commit 持续更新。

---

## 约定

### C-001: 示例约定

- **主题**: 代码命名规范
- **约束**: Controller 必须以 `Controller` 结尾，Service 必须以 `Service` 结尾
- **场景**: 所有 Java 类定义
- **来源**: 项目初始化定义

---

## 踩坑

### G-001: 示例踩坑记录

- **问题**: 描述遇到的问题现象
- **原因**: 问题产生的根本原因
- **解决**: 如何解决该问题
- **来源**: 关联的需求或任务编号

---

## 模式

### P-001: 示例模式

- **名称**: 模式名称
- **描述**: 模式的详细描述和使用场景
- **示例**:
  ```java
  // 代码示例
  ```
- **来源**: 从哪个项目或需求中提取

---

## 新增知识记录指引

### 添加约定
1. 分配递增 ID (C-XXX)
2. 明确主题、约束、适用场景
3. 标注来源（需求编号或任务编号）

### 添加踩坑
1. 分配递增 ID (G-XXX)
2. 清晰描述问题现象
3. 分析根本原因
4. 提供解决方案
5. 标注来源便于追溯

### 添加模式
1. 分配递增 ID (P-XXX)
2. 命名模式
3. 描述使用场景
4. 提供可运行的代码示例
5. 标注来源和参考链接
