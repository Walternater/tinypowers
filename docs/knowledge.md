# 领域知识库

> 项目级领域知识，`/tech:code` 编码时自动加载。
> 只记录模型无法从公开资料获取的内容：内部组件用法、平台约束、踩坑经验。
> 判断标准：如果你需要翻内部文档才能回答的问题，就属于领域知识。

## 使用说明

适合写入这里的内容：

- 只在当前项目或当前组织里成立的实现约束
- 重复踩过一次就值得避免的非显而易见坑
- 后续 feature 会直接复用的决策经验
- 从 `notepads/learnings.md` 中标记为 `[PERSIST]` 的高价值条目

不适合写入这里的内容：

- Google 或官方文档能直接查到的通用知识
- 只对单个 feature 临时有意义的执行笔记
- 还没有验证过的猜测
- 为了“看起来完整”而复制粘贴的背景说明

推荐做法：

- 保持短条目，优先写“场景 / 约束 / 结论”
- 从 feature 级 `learnings.md` 择优提升，不自动整段搬运
- 没有真正新知识时，宁可不写
- 初始化阶段优先从 `README.md` 和项目现有代码里提炼关键选型
