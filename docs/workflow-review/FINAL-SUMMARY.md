## 🎯 关键特性

### 1. 全自动流程

**之前**: 人工提取 → 人工阅读 → 人工复制 → 人工粘贴 → 人工整理

**现在**: AI 提取 → **AI 自动合并到主库** → 人工确认（单点）

### 2. 智能处理

| 功能 | 实现 |
|------|------|
| **去重** | 相似度 > 80% 自动跳过 |
| **分类** | 自动归入设计模式/架构决策/最佳实践/风险 |
| **合并** | 相似知识自动补充细节 |
| **备份** | 每次更新前自动备份 |
| **溯源** | 保留来源文件和添加时间 |

### 3. 人工干预点

```
AI 自动完成 90% 的工作
        ↓
人工确认（单点）
   ├── [y] 确认（默认，90% 情况）
   ├── [e] 编辑（10% 情况微调）
   ├── [r] 恢复（极少数回退）
   └── [n] 跳过（特殊不更新）
```

## 🛡️ 安全保障

### 1. 自动备份
- 每次更新前自动备份到 `.tmp/knowledge-backup/`
- 保留最近 20 个版本
- 可按时间戳恢复

### 2. 保守策略
- 相似度 > 80% 自动跳过（避免重复）
- 不确定时选择添加而非替换
- 保留所有原始提取记录

### 3. 人工兜底
- 最后一步必须确认
- 支持编辑修改
- 支持一键恢复

## 📝 配置选项

```bash
# 全自动模式（默认）
/tech:commit
# → AI 自动提取 + 自动合并 + 等待确认

# 半自动模式（提取但不自动合并）
AUTO_CONSOLIDATE=0 /tech:commit
# → AI 提取到 docs/auto/，人工手动合并

# 跳过知识提取
EXTRACT_KNOWLEDGE=0 /tech:commit

# 全量分析
ANALYSIS_MODE=full /tech:commit
```
