# AI 自动知识沉淀方案

> **人工确认 → AI 自动沉淀**

## 核心变化

### 之前：人工沉淀
```
AI 提取知识 → 人工阅读 → 人工复制粘贴 → 人工整理 → docs/knowledge.md
              ↑_______________________________________________|
                              （耗时、易错、负担重）
```

### 现在：AI 自动沉淀
```
AI 提取知识 → AI 自动分析 → AI 智能合并 → docs/knowledge.md → 人工确认即可
                                                  ↑
                                         （默认接受，有问题再改）
```

## 自动化程度

| 步骤 | 之前 | 现在 |
|------|------|------|
| **代码分析** | AI 自动 | AI 自动 ✅ |
| **知识提取** | AI 自动 | AI 自动 ✅ |
| **去重检测** | 人工判断 | AI 自动 ✅ |
| **分类整理** | 人工操作 | AI 自动 ✅ |
| **合并到主库** | 人工复制粘贴 | AI 自动 ✅ |
| **备份原文件** | 无 | AI 自动 ✅ |
| **最终确认** | 每一步都要确认 | 最后一步确认 ✅ |

## 新的工作流程

### 开发者视角（极简）

```bash
# 1. 开发代码
# ... 写代码 ...

# 2. 提交（全自动知识沉淀）

# [e] 编辑（如有问题）
# [r] 恢复（如不满意）
```

## 技术实现

### 新增组件

| 组件 | 文件 | 作用 |
|------|------|------|
| **收集脚本** | `collect-code-for-analysis.js` | 收集变更代码信息 |
| **合并脚本** | `ai-knowledge-consolidator.js` | 准备合并提示，触发 AI 合并 |

### AI 合并提示词示例

```markdown
你是一位知识管理专家，请将新提取的代码知识智能合并到主知识库中。

## 合并规则

### 1. 去重规则
- 计算新知识与现有知识的相似度
- >80% 相似：跳过（已存在）
- 50-80% 相似：合并补充细节
- <50% 相似：作为新条目添加

### 2. 分类规则
- 包含 "模式/Pattern" → 设计模式
- 包含 "决策/Decision/选择" → 架构决策
- 包含 "实践/建议/推荐" → 最佳实践
- 包含 "风险/问题/注意" → 已知问题与风险

### 3. 格式规则
每个知识点统一格式：

### [序号]. [标题]
**来源**: {文件路径}  
**添加时间**: {date}  
**描述**: ...  
**代码示例**: （如有）  
**价值/影响**: ...

### 4. 排序规则
- 保持现有知识顺序不变
- 新知识添加到各类别末尾
- 按添加时间排序

## 输入

### 现有知识库
[docs/knowledge.md 内容]

### 新提取的知识
[docs/auto/knowledge-{date}.md 内容]

## 输出

输出合并后的完整 docs/knowledge.md 内容：
```

## 安全保障

### 1. 自动备份
- 每次更新前自动备份到 `.tmp/knowledge-backup/`
- 保留最近 20 个版本
- 可随时恢复

### 2. 默认保守
- 相似度 > 80% 自动跳过（避免重复）
- 不确定时选择添加而非替换
- 保留所有历史版本在 `docs/auto/`

### 3. 人工兜底
- 最后一步必须确认
- 支持编辑修改
- 支持一键恢复

## 配置选项

```bash
# 全自动模式（默认）
/tech:commit
# → AI 自动合并，显示摘要，等待确认

# 半自动模式（只提取不合并）
AUTO_CONSOLIDATE=0 /tech:commit
# → AI 提取到 docs/auto/，人工手动合并

# 跳过知识提取
EXTRACT_KNOWLEDGE=0 /tech:commit

# 全量分析（项目总结）
ANALYSIS_MODE=full /tech:commit
```

## 故障恢复

### 场景 1: 合并后的知识库格式乱了
```bash
# 查看最新备份
ls -la .tmp/knowledge-backup/

# 恢复上一版本
cp .tmp/knowledge-backup/knowledge-backup-xxxxx.md docs/knowledge.md

# 或者恢复上上次
cp .tmp/knowledge-backup/knowledge-backup-yyyyy.md docs/knowledge.md
```

### 场景 2: 误删了重要知识
```bash
# 从备份中找回
grep "关键词" .tmp/knowledge-backup/*.md

# 手动合并回去
```

### 场景 3: AI 合并不准确
