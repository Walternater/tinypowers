## 智能化增强

### AI自动修复

#### 自动修复流程

```
1. 问题识别
   ↓
2. 修复方案生成
   ↓
3. 方案验证（编译/测试）
   ↓
4. 用户确认
   ↓
5. 应用修复
```

#### 支持场景

| 场景 | 检测来源 | 自动修复能力 |
|------|----------|--------------|
| 命名规范问题 | compliance-reviewer | ✅ 自动重命名 |
| 格式化问题 | check-style | ✅ 自动格式化 |
| 简单安全漏洞 | security-check | ✅ 自动修复（如添加校验） |
| 缺失依赖 | compile error | ✅ 自动添加 import |
| 测试骨架缺失 | coverage check | ✅ 自动生成 |
| 复杂逻辑问题 | code-review | ❌ 仅建议，需人工 |

#### 配置

```yaml
# .tinypowers/config.yaml
ai_fix:
  enabled: true
  auto_apply:
    - naming
    - formatting
    - import
  require_confirm:
    - security
    - logic
  max_fixes_per_run: 10
```

### 智能建议

#### 建议类型

1. **架构建议**
   - 检测到重复代码 → 建议提取公共方法
   - 检测到循环依赖 → 建议重构包结构
   - 检测到大数据量查询 → 建议添加分页

2. **性能建议**
   - 检测到 N+1 查询 → 建议使用 JOIN/Fetch
   - 检测到同步阻塞 → 建议使用异步
   - 检测到重复计算 → 建议缓存

3. **安全建议**
   - 检测到敏感日志 → 建议脱敏
   - 检测到弱加密 → 建议升级算法
   - 检测到越权风险 → 建议鉴权检查

4. **最佳实践建议**
   - 检测到旧API使用 → 建议迁移到新API
   - 检测到缺失文档 → 建议添加注释
   - 检测到测试覆盖率低 → 建议补充测试

#### 建议输出格式

```markdown
## 智能建议

### 建议-001: 提取重复代码
**位置**: UserService.java:45, OrderService.java:32
**问题**: 重复的数据转换逻辑
**建议**: 提取到 BaseService.convertData()
**置信度**: 85%
**操作**: [查看详情] [应用修复] [忽略]

### 建议-002: 添加分页
**位置**: OrderRepository.findAll()
**问题**: 可能返回大量数据
**建议**: 添加 Pageable 参数
**置信度**: 90%
**操作**: [查看详情] [应用修复] [忽略]
```
