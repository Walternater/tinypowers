# tinypowers 完整功能规格 (1.0 - 3.0+)

## 文档说明

本文档定义 tinypowers 从 1.0 MVP 到 3.0+ 的完整功能规格，每个版本包含：
- 版本目标
- 功能清单（按优先级）
- 交付物
- 验收标准
- 不做清单（明确排除）

---

## 1.0 MVP - 四技能框架跑通

### 版本目标
建立四技能 (/init /feature /code /commit) 框架，Java 场景完整跑通，明确定义与 superpowers 的边界。

### 架构定位
```
tinypowers（薄编排层）              superpowers（技能执行层）
─────────────────────────────────────────────────────────────
/tech:init ───────────────────────────────── (tinypowers 独有)
/tech:feature ─────────────────────────────→ brainstorming → writing-plans
/tech:code ────────────────────────────────→ worktrees → subagent → review → verification
/tech:commit ──────────────────────────────→ (独有) → finishing-branch
```

### 功能清单

#### P0 - /tech:init（独有）
| ID | 功能 | 说明 |
|----|------|------|
| 1.0-P0-001 | 技术栈检测 | 识别 pom.xml/build.gradle |
| 1.0-P0-002 | 骨架初始化 | 创建 CLAUDE.md、目录结构 |
| 1.0-P0-003 | 版本检查 | 本地 vs 远端版本对比 |
