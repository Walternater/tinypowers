## 1. 脚本接口

### 1.1 detect-stack.sh

**路径**: `scripts/detect-stack.sh`  
**用途**: 检测项目技术栈

**输出格式** (JSON):
```json
{
  "stack": "java",
  "buildTool": "maven",
  "framework": "spring-boot",
  "version": "3.2.0"
}
```

**返回值**:
- `0`: 检测成功
- `1`: 未检测到支持的构建工具

**错误输出** (stderr):
```
ERROR: 未检测到支持的构建工具 (需要 pom.xml 或 build.gradle)
```

---

### 1.2 check-gate-1.sh

**路径**: `scripts/check-gate-1.sh`  
**用途**: CHECK-1 门禁检查 (feature → code)

**输入参数**:
- `$1`: feature目录路径 (如 `features/FEAT-001-login`)

**输出格式**:
```
## CHECK-1 门禁检查

□ PRD.md 存在且非空 ... PASS
□ spec.md 存在且非空 ... PASS
□ tasks.md 存在且非空 ... PASS

结论: PASS
```

**返回值**:
- `0`: 通过
- `1`: 失败

### 1.5 pattern-scan.sh

**路径**: `scripts/pattern-scan.sh`  
**用途**: 扫描项目代码模式

**输入参数**:
- `$1`: 项目根目录 (可选，默认为当前目录)

**输出**: `patterns.md` 文件

**输出格式**:
```markdown
# 项目代码模式

## Controller 模式
- 命名风格: PascalCase
- 注解风格: @RestController + @RequestMapping
- 路径前缀: /api/v1

## Service 模式
- 命名风格: PascalCase + Service后缀
- 事务注解: @Transactional 在类级别

## Repository 模式
- 继承: JpaRepository
- 方法命名: findBy* / save* / delete*
```

**返回值**:
- `0`: 成功
- `1`: 失败

## 2. 文件格式契约

### 2.1 PRD.md 格式

**必须包含章节**:
```markdown
# PRD: {需求名称}

## 背景
{至少1段}

## 范围
### 包含
- {功能点1}
- {功能点2}

### 排除
- {明确不做}

## 验收标准
- [ ] AC-001: Given {上下文} When {动作} Then {结果}
- [ ] AC-002: ...
- [ ] AC-003: ...
```

**验收标准数量**: ≥3条  
**格式**: EARS (Given/When/Then)

---

### 2.2 spec.md 格式

**必须包含章节**:
```markdown
# 技术方案: {需求名称}

## 目标
{要达成什么}

## 核心设计
{架构描述}

## 锁定决策
| ID | 决策 | 理由 |
|----|------|------|
| D-001 | {关键决策} | {为什么} |
```

**锁定决策数量**: ≥1条  
**决策ID格式**: D-XXX (三位数字)

---

### 2.3 tasks.md 格式

**必须包含章节**:
```markdown
# 任务拆解: {需求名称}

| ID | 任务 | 验收标准 | 依赖 |
|----|------|---------|------|
| T-001 | {任务1} | {标准} | - |
| T-002 | {任务2} | {标准} | T-001 |
```

**任务数量**: ≤8个  
**任务ID格式**: T-XXX (三位数字)

---

### 2.4 VERIFICATION.md 格式

**必须包含章节**:
```markdown
# 验证报告: {需求名称}

## 验证结果
- [x] AC-001: {验收标准} → PASS
- [x] AC-002: ... → PASS
- [ ] AC-003: ... → FAIL (原因)

## 决策落地检查
- [x] D-001: {决策} → {代码位置}

## 结论
**PASS** / **FAIL**

## 备注
{其他信息}
```

---

### 2.5 SPEC-STATE.md 格式

**内容**:
```markdown
# 需求状态

**需求ID**: FEAT-001
**当前状态**: PLAN | EXEC | REVIEW | DONE
**最后更新**: 2026-04-09

## 状态流转记录
- 2026-04-09: PLAN → EXEC (用户确认进入开发)
```

**有效状态**: PLAN, EXEC, REVIEW, DONE

## 4. Agent 接口契约

### 4.1 compliance-reviewer

**输入**:
- `spec.md` 路径
- 代码 diff（git diff）

**输出格式**:
```markdown
## Compliance Review 结果

### 决策落地
- [x] D-001: 已落地 → OrderController.java:45
- [ ] D-002: 未找到对应实现 ⚠️

### 接口符合
- [x] API 路径符合
- [x] 参数定义符合

### 数据符合
- [x] DB 变更符合

### 范围符合
- [x] 无范围外变更

### 安全符合
- [x] 无安全风险

## 结论
**BLOCK**: 1 / **WARN**: 0 / **PASS**: 4
```

**结论规则**:
- BLOCK > 0: 必须修复
- WARN > 0, BLOCK = 0: 可继续但建议修复
- BLOCK = 0, WARN = 0: 通过

## 5. 版本兼容性承诺

### 5.1 向后兼容

v1.0 承诺以下接口在 1.x 系列中保持兼容:
- 所有脚本接口 (输入/输出/返回值)
- 所有文件格式 (PRD.md, spec.md, tasks.md, VERIFICATION.md)
- SPEC-STATE.md 状态定义

### 5.2 不兼容变更

以下变更需要主版本升级 (2.0):
- 新增状态 (如添加 TESTING)
- 脚本接口变更 (新增必填参数)
- 文件格式重大变更 (如 YAML 替代 Markdown)

## 6. 契约变更流程

### 6.1 变更类型

| 类型 | 说明 | 处理 |
|------|------|------|
| PATCH | 文档错误修正 | 直接修改 |
| MINOR | 新增可选字段 | 协商后修改 |
| MAJOR | 接口/格式变更 | 冻结后不可变更 |

### 6.2 冻结后变更

一旦标记为 **FROZEN**，本契约不可变更。如需变更:
1. 创建新版本契约 (如 v1.0.1)
2. 在相关版本文档中说明变更
3. 更新依赖关系
