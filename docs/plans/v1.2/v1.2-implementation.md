### 规则库（50条）

```markdown
## compliance-reviewer 规则库 v1.2

### 命名规范（10条）
- [N001] 类名使用 PascalCase
- [N002] 方法名使用 camelCase
- [N003] 常量使用 UPPER_SNAKE_CASE
- [N004] 包名使用小写
- [N005] 接口名不以 I 开头
- [N006] 实现类以 Impl 结尾（可选）
- [N007] 测试类以 Test 结尾
- [N008] 异常类以 Exception 结尾
- [N009] 枚举使用 PascalCase，值使用 UPPER_SNAKE_CASE
- [N010] 布尔方法以 is/has/should 开头

### 代码结构（15条）
- [S001] Controller 只处理 HTTP 相关
- [S002] Service 包含业务逻辑
- [S003] Repository 只访问数据
- [S004] Entity 只定义数据结构
- [S005] DTO 用于层间传输
- [S006] 不直接使用 Repository 在 Controller
- [S007] Service 方法有事务注解（如需）
- [S008] 公共方法有 JavaDoc
- [S009] 复杂逻辑有注释
- [S010] 不使用魔法数字
- [S011] 字符串常量提取
- [S012] 不使用 System.out.println
- [S013] 日志使用 SLF4J
- [S014] 异常不吞没，至少记录
- [S015] 资源使用 try-with-resources

### 安全检查（20条）
- [SEC001] SQL 使用参数化查询
- [SEC002] 用户输入校验长度
- [SEC003] 用户输入校验类型
- [SEC004] 敏感信息不输出到日志
- [SEC005] 敏感信息不返回给前端
- [SEC006] 密码不明文存储
- [SEC007] 使用 HTTPS
- [SEC008] Cookie 设置 HttpOnly
- [SEC009] 防 XSS：输出编码
- [SEC010] 防 CSRF：Token 验证
- [SEC011] 文件上传限制类型
- [SEC012] 文件上传限制大小
- [SEC013] 敏感操作需要鉴权
- [SEC014] 不使用不安全的反序列化
- [SEC015] 依赖无已知漏洞
- [SEC016] 不暴露内部错误详情
- [SEC017] 使用安全的随机数
- [SEC018] 会话有超时
- [SEC019] 防重放攻击（关键操作）
- [SEC020] 敏感数据加密存储

### 性能检查（5条）
- [PERF001] 循环内不查询 DB
- [PERF002] 大数据量查询分页
- [PERF003] 不使用 SELECT *
- [PERF004] 批量操作使用批量接口
- [PERF005] 异步处理耗时操作
```

**审查流程**：
```
1. 开发者决策自查
   ↓
2. compliance-reviewer（tinypowers）
   - 方案符合性
   - 决策落地
   - 50+ 规则检查
   ↓ 通过
3. requesting-code-review（superpowers）
   - 代码质量
   - 设计模式
   - 性能
   ↓ 通过
4. verification（superpowers）
   ↓ 通过
5. CHECK-2 通过
```
