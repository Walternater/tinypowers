# bootstrap-guard.md

## 作用

这份文档定义 `/tech:init` 在进入主流程前的预检规则。

预检的目标不是暴露更多流程步骤，而是在主流程开始前先处理边界场景，避免产出错误的初始化结果。

## 处理原则

- 预检不占用主流程编号
- 预检默认非交互执行
- 一旦命中高风险场景，优先短路退出，而不是带着错误前提继续初始化
- 预检命中时，要明确告诉用户“为什么没有进入主流程”

## 预检场景

### 1. 框架仓库自举保护

在执行任何目标项目初始化动作前，先判断当前项目是否是 tinypowers 框架自身。

判断条件建议沿用旧逻辑：满足任意两个即可视为框架仓库。

- 存在 `skills/` 目录且包含 `SKILL.md`
- 存在 `agents/` 目录且包含多个 `.md` 文件
- 存在 `manifests/components.json`
- 存在 `hooks/` 目录

如果判定为框架仓库：

- 不进入普通项目初始化主流程
- 不创建目标项目导向的 `.claude/hooks/`
- 不生成目标项目导向的 `.claude/settings.json`
- 不创建面向业务项目的 `CLAUDE.md`、`features/`、`docs/knowledge.md`
- 允许保留只读诊断或提示信息

推荐提示：

```text
检测到当前目录是 tinypowers 框架仓库。
已跳过目标项目初始化产物，避免框架对自身重复执行 /tech:init。
```

### 2. 非支持技术栈

如果检测到当前项目不是 Java 项目，则不要进入主流程。

推荐行为：

- 输出当前识别到的项目特征
- 明确说明 `/tech:init` 当前只支持 Java
- 建议用户暂不初始化，或等待对应规则集补齐

### 3. 初始化资源缺失

如果初始化所需基础资源缺失，也应在预检阶段停止，例如：

- 找不到 hooks 源目录
- 找不到 `configs/templates/settings.json`
- 找不到 `configs/templates/CLAUDE.md`

推荐行为：

- 不执行半套初始化
- 明确提示安装不完整或路径有误
- 引导用户先修复安装，再重新执行 `/tech:init`

## 输出要求

预检命中时，输出应满足：

- 一句话说明命中了哪个 guard
- 一句话说明为什么停止
- 一句话说明下一步建议

不要把 guard 的内部细节直接塞回主流程说明里。
