# task-breakdown.md

## 任务拆解（Scope → Stories）

本文档描述如何将需求分解为可执行任务。

---

## 分层结构

```
Scope（范围）
  ↓ 一个需求是一个 Scope
Epic（史诗）
  ↓ 分解为一个或多个 Epic
Story（故事）
  ↓ 分解为一个或多个 Task
Task（任务）
```

---

## 分解原则

### Epic 分解

| 原则 | 说明 |
|------|------|
| 内聚 | 同一 Epic 的功能高相关 |
| 可独立测试 | Epic 可以独立验收 |
| 2-4周 | 一个 Epic 在 2-4 周完成 |

### Story 分解

| 原则 | 说明 |
|------|------|
| 独立可开发 | 不依赖其他 Story |
| 独立可测试 | 有明确的验收标准 |
| 1-3天 | 一个 Story 在 1-3 天完成 |

### Task 分解

| 原则 | 说明 |
|------|------|
| 2-4小时 | 一个 Task 在 2-4 小时完成 |
| 有验收标准 | 可验证完成 |
| 明确类型 | 后端/前端/测试 |

---

## Story 验收标准

每个 Story 必须包含验收标准（Acceptance Criteria）：

```markdown
## Story 1.1: 用户名密码登录

**作为** 访客
**我想要** 使用用户名密码登录
**以便** 安全访问我的账户

### 验收标准

| # | 标准 | 测试场景 |
|---|------|----------|
| 1 | 输入正确账号密码，登录成功 | TC-001 |
| 2 | 输入错误密码，提示"密码错误" | TC-002 |
| 3 | 输入不存在用户，提示"用户不存在" | TC-003 |
| 4 | 连续5次失败，账户锁定15分钟 | TC-004 |
| 5 | 登录成功跳转首页 | TC-005 |

### 前后端职责

| 职责 | 前端 | 后端 |
|------|------|------|
| 登录页渲染 | ✅ | - |
| 表单校验 | ✅ | - |
| 登录接口调用 | ✅ | ✅ |
| 密码校验 | - | ✅ |
| 会话创建 | - | ✅ |
| 登录状态管理 | ✅ | ✅ |
```

## 估时方法

### T-Shirt 估时法

| 尺寸 | 实际时间 | 适用场景 |
|------|----------|----------|
| XS | 1-2h | 简单改动 |
| S | 0.5d | 1天内完成 |
| M | 1d | 标准任务 |
| L | 2d | 复杂任务 |
| XL | 3d+ | 需拆解 |

### 三点估时法

```
估时 = (乐观 + 4*最可能 + 悲观) / 6

示例：
- 乐观：0.5d
- 最可能：1d
- 悲观：2d
- 估时 = (0.5 + 4*1 + 2) / 6 = 1.08d ≈ 1d
```

## 依赖管理

### 依赖类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 技术依赖 | 必须先完成某任务 | DB设计→接口开发 |
| 业务依赖 | 必须先完成某功能 | 登录→下单 |
| 人力依赖 | 需要特定人完成 | 需要DBA协助 |

### 依赖处理

```
依赖处理原则：
1. 优先并行 — 充分利用人力
2. 识别关键路径 — 跟踪项目工期
3. 预留缓冲 — 复杂任务加20%缓冲
```
