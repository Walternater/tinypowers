# verification.md

## 各阶段验证清单

本文档描述 tech-feature 每个阶段的验证标准。

---

## Phase 0: 准备

| 验证项 | 验证方法 | 通过标准 |
|--------|----------|----------|
| 目录创建 | `test -d features/{id}` | 目录存在 |
| 分支创建 | `git branch --list feature/{id}` | 分支存在 |
| 目录结构 | `ls features/{id}/` | 至少包含 PRD.md |

---

## Phase 1: 需求理解

| 验证项 | 验证方法 | 通过标准 |
|--------|----------|----------|
| PRD 存在 | `test -f features/{id}/PRD.md` | 文件存在 |
| PRD 可读 | 读取文件内容 | 能提取功能点 |
| 功能点提取 | 分析 PRD | 至少3个功能点 |

---

## Phase 2: 歧义检测

| 验证项 | 验证方法 | 通过标准 |
|--------|----------|----------|
| 模糊点识别 | 检查 PRD | 无高优先级歧义 |
| 澄清问题 | 生成问题列表 | 问题有建议定义 |
| 需求确认 | 用户确认 | 无阻塞性问题 |

---

## Phase 3: 技术方案

| 验证项 | 验证方法 | 通过标准 |
|--------|----------|----------|
| 方案存在 | `test -f features/{id}/技术方案.md` | 文件存在 |
| 架构清晰 | 读取内容 | 有架构图和说明 |
| 接口完整 | 检查接口设计 | 覆盖所有功能 |
| 自检通过 | 对照检查清单 | 全部通过 |

---

## Phase 4: 任务拆解

| 验证项 | 验证方法 | 通过标准 |
|--------|----------|----------|
| 拆解表存在 | `test -f features/{id}/任务拆解表.md` | 文件存在 |
| Epic 完整 | 检查 Epic | 覆盖所有功能 |
| Story 完整 | 检查 Story | Epic 下有 Story |
| Task 完整 | 检查 Task | Story 下有 Task |
| 验收标准 | 检查每个 Story | 都有验收标准 |
| 估时合理 | 检查总工时 | 与预期相符 |
| 依赖明确 | 检查依赖 | 无循环依赖 |

---

---

## 失败处理

| 阶段 | 失败原因 | 处理方式 |
|------|----------|----------|
| Phase 0 | 目录创建失败 | 提示权限问题 |
| Phase 1 | PRD 不存在 | 暂停等待用户创建 |
| Phase 2 | 高优先级歧义 | 必须澄清才能继续 |
| Phase 3 | 方案评审不通过 | 修改方案后重新评审 |
| Phase 4 | 拆解不完整 | 补充拆解后继续 |
