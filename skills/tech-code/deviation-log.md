# Deviation Log

记录执行过程中自动修复的偏差。

## 格式

```markdown
## YYYY-MM-DD HH:MM

| 时间 | 任务ID | 规则 | 描述 | 证据 |
|------|--------|------|------|------|
| HH:MM | T-XXX | Rule N | 描述 | 文件:行号 |
```

## 规则说明

| 规则 | 条件 | 动作 |
|------|------|------|
| Rule 1 | 发现 Bug | 自动修复，不询问 |
| Rule 2 | 发现遗漏的关键功能 | 自动补全，不询问 |
| Rule 3 | 遇到阻塞问题 | 自动解决阻塞，不询问 |
| Rule 4 | 架构变更 | 暂停，询问用户 |

## 示例

```markdown
## 2026-03-30 10:30

| 时间 | 任务ID | 规则 | 描述 | 证据 |
|------|--------|------|------|------|
| 10:30 | T-042 | Rule 1 | 修复空指针异常 | src/utils.js:42 |
| 10:35 | T-042 | Rule 2 | 补全缺失的参数校验 | src/utils.js:28 |
```
