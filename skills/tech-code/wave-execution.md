# wave-execution.md

## 作用

把任务表按依赖推进分 Wave：能并行的并行，必须串行的串行。

## 前置条件

`tech-plan-checker` 已通过、锁定决策已回顾、STATE.md 已创建、SPEC-STATE.md phase 为 `TASKS` 或 `EXEC`。

## 1. 构建依赖图

根据 `任务拆解表.md` 建立任务依赖关系。检查：循环依赖、遗漏前置、本可并行却串成链的任务。

## 2. 分解成 Wave

当前无未完成前置依赖的任务进入同一 Wave，全部完成后才启动下一 Wave。

```text
Wave 1: T-001
Wave 2: T-002, T-003
Wave 3: T-004
```

## 3. 分配任务

同一 Wave 内可并行分配给独立 Agent。每个任务 prompt 必须包含：任务描述、验收标准、相关文件、依赖接口、决策 ID。

强制要求：先读再改、验收标准可验证、禁止占位实现。

## 4. 等待 Wave 收敛

所有任务返回结果、失败任务已处理、STATE.md 已更新、质量门禁已执行。

## 5. Wave 末尾质量门禁

编译 → 单元测试 → 覆盖率 → 安全扫描。命令从 CLAUDE.md `build_command` 读取，标准见 `quality-gate.md`。

## 6. 失败处理

- 任务级：定位实现/依赖/方案问题，可修复则继续，需重排则更新任务表
- Wave 级：阻断下一 Wave，修复后重跑门禁

## 7. 3 次失败升级

同一问题连续 3 次失败后停止同方向尝试，转入 `deviation-handling.md` 架构质疑流程。

## 8. STATE.md 记录

每个 Wave 同步：当前 Wave 编号、已完成/失败任务、门禁结果、上次操作摘要。

## 9. 完成判断

Wave 执行完成表示所有任务按依赖落地、每个 Wave 通过门禁，可进入后续顺序审查。
