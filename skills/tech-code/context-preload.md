# context-preload.md

## 作用

这份文档定义 `/tech:code` 在 Wave Execution 阶段如何通过上下文预加载减少重复文件读取，避免每个 subagent 独立读同一套文件浪费 context。

## 核心原则

- 编排层读一次，传递给所有 subagent
- subagent 收到预加载的上下文后，不应重新读取
- 预加载的内容只包含当前任务需要的最小集合

## 预加载协议

### Step 1: 编排层收集上下文

在启动任何 Wave 之前，编排层一次性读取以下文件：

```text
必读：
  features/{id}/技术方案.md          # 技术方案全文
  features/{id}/任务拆解表.md        # 任务表全文
  features/{id}/SPEC-STATE.md        # 当前进度

按需读（如果存在）：
  features/{id}/STATE.md             # 执行状态
  features/{id}/需求理解确认.md       # 需求确认
  CLAUDE.md                          # 项目入口
```

### Step 2: 注入到任务 prompt

每个 subagent 的任务 prompt 中，在具体任务描述之前，插入预加载区块：

```text
--- 预加载上下文 START ---

## 技术方案摘要

{从 技术方案.md 提取的核心内容}

## 当前任务上下文

- Feature: {id}
- 当前 Wave: {wave_number}
- 已锁定决策: {从 技术方案.md 决策表提取}
- 已完成 Wave: {从 STATE.md 提取}

## 相关接口 / 数据结构

{从 技术方案.md 提取与当前任务相关的接口定义和数据结构}

--- 预加载上下文 END ---

重要：以上上下文已预加载，你不需要重新读取这些文件。
只读取你需要修改的目标文件。

{任务具体描述开始...}
```

### Step 3: subagent 的读取约束

subagent 收到预加载上下文后：
- 禁止重新读取已预加载的文件
- 只读取需要修改的目标源码文件
- 如果预加载内容与实际文件有冲突，以实际文件为准并报告偏差

## 裁剪规则

预加载不是全量塞入。按以下规则裁剪：

| 内容类型 | 裁剪规则 |
|---------|---------|
| 技术方案全文 | 只注入与当前 Wave 任务相关的章节 |
| 接口定义 | 只注入当前任务涉及的接口 |
| 数据库设计 | 只注入当前任务涉及的表 |
| 决策记录 | 全量注入（决策是全局约束） |
| 需求理解 | 只注入核心摘要，不超过 20 行 |

## 适用范围

- **适用**：Wave Execution 中的 subagent 任务分发
- **不适用**：审查阶段的 agent（审查需要独立读取完整文件）
- **不适用**：tech-feature 的各 Phase（每个 Phase 需要不同的上下文深度）

## 与审查阶段的关系

审查阶段（spec-compliance-reviewer、security-reviewer、code-reviewer）必须独立读取文件，不能依赖预加载上下文。原因是：
- 审查的独立性要求：审查者不能信任被审查者提供的上下文
- 审查需要看完整文件，不是摘要
- 预加载可能遗漏审查者需要的细节
