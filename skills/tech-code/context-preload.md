# context-preload.md

## 作用

定义 `/tech:code` 在 Wave Execution 阶段的上下文预加载规则。

## 预加载流程

### 编排层读取（每个 Wave 开始前）

一次性读取共享文档：

- `features/{id}/技术方案.md`
- `features/{id}/任务拆解表.md`
- `features/{id}/SPEC-STATE.md`
- `features/{id}/STATE.md`
- `features/{id}/notepads/learnings.md`（如果存在）

### 裁剪规则

| 内容类型 | 裁剪策略 | 理由 |
|---------|---------|------|
| 技术方案全文 | 只注入当前任务相关章节 | 避免全量注入 |
| 接口定义 | 只注入当前任务涉及的接口 | 按任务裁剪 |
| 数据库设计 | 只注入当前任务涉及的表 | 按任务裁剪 |
| 决策记录 | 全量注入 | 决策是全局约束 |
| 规则文件 | 不注入（由 Hook 加载） | Hook 自动处理 |

### subagent 执行

每个 subagent 任务 prompt 必须包含：
- 任务描述和验收标准
- 相关文件路径和技术方案片段
- 已锁定的决策
- **参考实现文件内容**（来自 Pattern Scan）

subagent 只读取 prompt 中指定的目标文件，不重复读取已预加载的文档。

### 参考代码预加载（缝合扫描后）

根据 Pattern Scan 结果，为非 GREENFIELD 任务预加载参考实现：
- 读取参考文件的完整内容
- 在 prompt 中标注哪些部分作为骨架保留（目录结构、错误处理模式、请求封装）
- 标注哪些部分需要替换（业务字段、接口地址、数据模型）

```text
参考文件: src/user/UserController.java
保留: 类注解结构、统一响应封装、参数校验模式
替换: 路由路径 /api/users → /api/auth/login, User → LoginDTO
新增: JWT token 生成逻辑（UserController 无此模式）
```

### 领域知识预加载

在参考代码预加载之后，加载项目级领域知识。

**来源**：项目根目录的 `docs/knowledge.md`（如果不存在则跳过，不阻断流程）

**加载方式**：按任务相关性裁剪注入。

| 知识类型 | 裁剪策略 | 理由 |
|---------|---------|------|
| 组件用法 | 只注入当前任务涉及的组件 | 避免无关组件干扰 |
| 平台约束 | 全量注入 | 约束是全局底线 |
| 踩坑记录 | 只注入当前任务涉及的技术领域 | 按相关性过滤 |

**判断标准**：只记录模型无法从公开资料获取的知识：
- 内部组件 X 的特殊用法？→ 不能从公开资料获取 → 需要记录
- 接口地址必须以 `/` 开头否则被网关拦截？→ 不能 → 需要记录
- React useState 怎么用？→ 能 → 不需要
- tsconfig 配置报错怎么解？→ 能 → 不需要

## 适用范围

- **适用**：Wave Execution 阶段
- **不适用**：审查阶段（审查需要独立读取完整文件）
