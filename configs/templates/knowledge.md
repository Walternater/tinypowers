## 🤖 AI 自动知识沉淀

本项目已集成 **AI 驱动的全自动知识沉淀**。

### 工作方式

```
代码提交
    │
    ├── 1. 增量分析变更代码
    ├── 2. AI 提取设计模式、架构决策、潜在风险
    ├── 3. AI 自动合并到本文档（智能去重分类）
    └── 4. 显示变更摘要，等待确认
```

### 全自动流程

每次执行 `/tech:commit` 时：

1. **自动分析**: 检测本次变更的代码文件
2. **自动提取**: AI 识别新增的设计模式、架构决策
3. **自动沉淀**: AI 智能合并到本文档，自动去重分类
4. **等待确认**: 显示变更摘要，默认自动继续

```bash
/tech:commit

输出:
  🧠 AI 知识沉淀完成:
     ✅ 新增设计模式: 状态机模式
     ✅ 新增架构决策: 乐观锁替代悲观锁
     ⚠️  新增风险提醒: 并发超卖风险
     
     📄 docs/knowledge.md 已自动更新
     
  💡 请确认 [y/n/e]: 
     y - 确认无误（默认）
     n - 跳过本次更新
     e - 编辑修改
```

### AI 智能合并规则

- ✅ **自动去重**: 相似度 > 80% 自动跳过
- ✅ **智能分类**: 自动归入设计模式/架构决策/最佳实践/风险
- ✅ **保留历史**: 记录添加时间和来源文件
- ✅ **增量更新**: 不影响已有知识，只追加新内容

### 备份与恢复

每次更新前自动备份：
```
.tinypowers/knowledge-backup/
├── knowledge-backup-2026-04-08T10-30-00.md
├── knowledge-backup-2026-04-09T14-20-15.md
└── ...
```

如需恢复：
```bash
# 查看最新备份
ls -la .tinypowers/knowledge-backup/

# 恢复上一版本
cp .tinypowers/knowledge-backup/knowledge-backup-xxxxx.md docs/knowledge.md
```

### 手动触发

```bash
# 分析并自动沉淀（默认）
/tech:commit

# 只分析，不自动沉淀（人工确认后再合并）
AUTO_CONSOLIDATE=0 /tech:commit

# 全量分析整个项目
ANALYSIS_MODE=full /tech:commit
```
