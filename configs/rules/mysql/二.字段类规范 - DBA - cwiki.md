# 二.字段类规范

> 本文件补充数据库侧约束，作为 `configs/rules/common/` 通用规则在 MySQL 场景下的扩展。

#### 【建议】合理使用各种数值类型

> 【注意】int(11)中的11并不是长度，而是宽度，该值不会影响所存储数字的最大值。最大值是由字段类型决定的，如下：
> 
> 1.TINYINT：很小的整数。带符号的范围是-128到127。无符号的范围是0到255。
> 
> 2.SMALLINT：小的整数。带符号的范围是-32768到32767。无符号的范围是0到65535。
> 
> 3.MEDIUMINT：中等大小的整数。带符号的范围是-8388608到8388607。无符号的范围是0到16777215。
> 
> 4.INT：普通大小的整数。带符号的范围是-2147483648到2147483647。无符号的范围是0到4294967295。
> 
> 5.BIGINT：大整数。带符号的范围是-9223372036854775808到9223372036854775807。无符号的范围是0到18446744073709551615。

#### 【建议】避免使用null字段

> 1.null很难进行查询优化
> 
> 2.null列加索引，需要额外空间
> 
> 3.含null的复合索引无效
> 
> 反例：‘a’ char(32) default null             
>      ‘b’ int(10) not null

#### 【建议】少用并拆分TEXT/BLOB类型字符串

> 1.能使用varchar类型的不要使用text，若必须使用则拆分到单独的表。会占用额外的临时表空间，性能杀手。
> 
> 2.varchar最大长度控制在254以内
> 
> 3.禁止在数据库中存储图片,blob字段，文件等
