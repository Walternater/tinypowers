# 通用安全规范

所有项目必须遵守的安全红线。

---

## 1. 注入防护

### SQL注入
- **必须**：使用参数化查询
- **禁止**：字符串拼接 SQL

```java
// 正确
@Query("SELECT * FROM task WHERE id = :id")
Task findById(@Param("id") Long id);

// 错误 - SQL注入风险
@Query("SELECT * FROM task WHERE id = " + id)
```

### 其他注入
- 命令注入：用户输入传入系统命令前必须验证
- 代码注入：避免执行不可信脚本

---

## 2. 敏感信息

### 禁止日志输出
- 密码、Token、身份证号
- 银行卡号、密钥

```java
// 禁止！
log.info("User password: {}", user.getPassword());
```

### 敏感数据存储
- 密码必须哈希存储
- 密钥使用环境变量或密钥管理器

---

## 3. 认证与授权

- [ ] 接口有必要的权限校验
- [ ] 使用 `@PreAuthorize` 等框架注解
- [ ] JWT 验证正确
- [ ] 敏感接口双重验证

---

## 4. 输入校验

- [ ] 必填参数 null 检查
- [ ] 字符串长度限制
- [ ] 格式校验（手机号、邮箱等）
- [ ] 文件上传类型限制

---

## 5. 依赖安全

- [ ] 定期更新依赖版本
- [ ] 使用 `npm audit` / `mvn dependency-check` 检查漏洞
- [ ] 禁止使用有已知漏洞的库

---
