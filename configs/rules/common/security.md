# 通用安全规范

本文件定义所有项目默认共享的安全红线。

它的职责不是描述完整安全体系，而是明确：
- 哪些行为默认禁止
- 哪些检查默认必须做
- 哪些风险一旦出现就应优先处理

## 基本原则

1. 默认不信任任何外部输入。
2. 敏感信息不出现在代码、日志、截图和提交记录里。
3. 接口默认需要认证和授权边界。
4. 有注入风险的地方必须参数化或白名单化。
5. 依赖升级和漏洞检查是持续动作，不是上线前临时补救。

## 注入防护

### SQL 注入

- 必须使用参数化查询
- 禁止字符串拼接 SQL

```java
// 正确
@Query("SELECT * FROM task WHERE id = :id")
Task findById(@Param("id") Long id);

// 错误
@Query("SELECT * FROM task WHERE id = " + id)
```

### 其他注入

- 命令注入：用户输入进入系统命令前必须验证
- 代码注入：避免执行不可信脚本或模板
- 路径注入：文件路径必须限制在允许范围内

## 敏感信息

### 日志红线

禁止输出：
- 密码
- Token
- 密钥
- 身份证号、银行卡号等敏感身份信息

```java
// 禁止
log.info("User password: {}", user.getPassword());
```

### 存储要求

- 密码必须哈希存储
- 密钥应放到环境变量或密钥管理系统
- 不要把测试密钥长期留在仓库中

## 认证与授权

默认检查：

- [ ] 接口有必要的权限校验
- [ ] 使用 `@PreAuthorize` 或等价机制
- [ ] Token / JWT 验证完整
- [ ] 敏感操作具备更严格的校验或二次确认

## 输入校验

默认检查：

- [ ] 必填参数校验
- [ ] 长度限制
- [ ] 格式校验
- [ ] 枚举值 / 状态值白名单
- [ ] 文件上传类型限制

## 依赖安全

默认检查：

- [ ] 定期更新依赖版本
- [ ] 使用 `npm audit`、`mvn dependency-check` 或同类工具检查漏洞
- [ ] 禁止继续使用已知高危漏洞组件

## 最小检查清单

提交前至少确认：

| 检查项 | 状态 |
|--------|------|
| SQL 使用参数化 | [ ] |
| 无硬编码密钥 | [ ] |
| 日志无敏感信息 | [ ] |
| 接口有权限校验 | [ ] |
| 依赖无已知高危漏洞 | [ ] |

## 适用边界

这份规则适合作为“所有项目的默认下限”。

如果项目有更高要求，比如：
- 金融级审计
- 多租户隔离
- 外网开放 API
- 文件上传 / 第三方回调 / SSRF 风险

应在技术栈规则或项目级规则中继续补充，而不是只依赖本文件。
